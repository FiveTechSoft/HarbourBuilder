hbcpp - C++ powered GUI for Harbour
=====================================

2026-04-01
----------
- Added Linux GTK3 backend
  * gtk3_core.c: full reimplementation of C++ core using GTK3
    GtkWindow, GtkFixed for absolute positioning, all controls
    (Label, Edit, Button, CheckBox, ComboBox, GroupBox)
  * Design mode with GtkOverlay + GtkDrawingArea + Cairo
    Drag, resize with 8 handles, multi-selection, arrow keys
  * gtk3_inspector.c: property inspector using GtkTreeView/GtkListStore
    Color picker via GtkColorChooserDialog
    Font picker via GtkFontChooserDialog
    Category collapse/expand via double-click
  * inspector_gtk.prg: Harbour wrapper for GTK3 inspector
  * test_design_gtk.prg: design-mode sample for Linux
  * build_gtk.sh: build script with auto-detection of Harbour paths
  * Same Harbour OOP layer (classes.prg, commands.ch) works unchanged
  * Prerequisite: sudo apt install libgtk-3-dev
  * Build and run:
      cd samples
      ./build_gtk.sh
      ./test_design_gtk

- Moved screenshot images to images/ folder
  * images/MacOS.png, images/gtk3.png, images/linux.png
  * Updated README.md to reference new paths

2026-03-31
----------
- Cocoa inspector: nClrPane editable with "..." button and NSColorPanel
  * Click on nClrPane value or "..." button opens macOS standard color picker
  * Color swatch shown as cell background in the value column
  * Continuous color updates applied in real-time via UI_SetProp

- Cocoa inspector: oFont editable with "..." button and NSFontPanel
  * Click on oFont value or "..." button opens macOS standard font panel
  * Font displayed as "FontName,Size", applied via UI_SetProp
  * Initial font pre-selected in the font panel from current value

- Cocoa inspector: category collapse/expand via single-click
  * Click on category header (Appearance, Position, etc.) toggles collapse
  * Shows "+" when collapsed, "-" when expanded

- Fixed NSPopUpButton (ComboBox) positioning on macOS
  * Win32 combobox height is dropdown height, not control height
  * NSPopUpButton uses fixed intrinsic height (26px) instead of passed height
  * Fixes mispositioned combobox at startup

- Added macOS Cocoa/AppKit backend
  * cocoa_core.m: full reimplementation of C++ core (TControl, TForm, all controls)
    using NSView/NSWindow with flipped coordinates (top-left origin like Win32)
  * cocoa_inspector.m: property inspector using NSTableView
    Exports: INS_Create, INS_RefreshWithData, INS_BringToFront, INS_Destroy
  * inspector_mac.prg: Harbour wrapper for Cocoa inspector
  * test_design_mac.prg: design-mode sample for macOS
  * build_mac.sh: build script using clang + Harbour for macOS
  * Same Harbour OOP layer (classes.prg, commands.ch) works unchanged

- Added nClrPane (background color) property to all controls
  * Form: changes background brush and design-mode grid
  * Controls: handled via WM_CTLCOLOR* messages from parent form
  * Buttons: uses BS_OWNERDRAW + WM_DRAWITEM for custom background
  * Inspector shows color value with "..." button to open ChooseColor dialog

- Added oFont property to all controls
  * Displays as "FontName,Size" in inspector
  * "..." button opens ChooseFont dialog
  * Form font change propagates to all children
  * Individual control font can be set independently

- Applied Hungarian notation to all property names
  * String: cName, cText, cClassName, cFontName
  * Number: nLeft, nTop, nWidth, nHeight, nFontSize, nItemIndex, nItemCount
  * Logical: lVisible, lEnabled, lTabStop, lCenter, lDefault, lCancel,
             lReadOnly, lPassword, lChecked
  * Color: nClrPane
  * Font: oFont

- Inspector: color/font picker button fix
  * Button subclassed with InsBtnProc to handle click directly
  * Fixes issue where WM_COMMAND was swallowed by parent ListView
  * WM_KILLFOCUS ignores focus transfer to "..." button

- Initial project structure with C++ core and Harbour bridge
  * TControl base class with property system
  * TForm with design mode (drag, resize, rubber band selection, arrow keys)
  * TLabel, TEdit, TButton, TCheckBox, TComboBox, TGroupBox controls
  * Property inspector with collapsible categories and live updates
  * Overlay-based selection handles (layered window, per-pixel alpha)
  * Multi-backend: Win32 GUI, Console TUI, Web/HTML5 Canvas
  * JSON serialization of form layouts
  * OOP Harbour wrappers (classes.prg) over C++ core
  * xBase-style command syntax (commands.ch)
  * Build system for BCC77C + Harbour
